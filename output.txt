1. CONFIRMED ISSUES
- **Area B: Generation pipeline (Core Engine Failure)**
  - **Issue:** Video generation breaks completely when using modern MoviePy versions (`v2.0.0+`) due to API changes (`AttributeError: 'VideoFileClip' object has no attribute 'crop'`).
  - **Files:** `requirements.txt`
  - **Functions:** Entire pipeline dependency resolution
  - **Root Cause:** Dependency was pinned as `<2.1` which permitted pulling `v2.0.0`, breaking the `v1.x`-centric `generador.py` logic.
  - **Severity:** CRITICAL
  - **Status:** FIXED (Pinned back to `<2.0.0`)

- **Area C: Provider integrations (ElevenLabs Failure)**
  - **Issue:** The ElevenLabs wrapper invocation passes `out_mp3` as the positional `api_key` argument, dropping the actual file writing process and silently corrupting the audio file path generation.
  - **Files:** `generador.py`
  - **Functions:** `_generate_tts`
  - **Root Cause:** Parameter position mismatch between the caller in `generador.py` and the signature in `voz_elevenlabs.py`.
  - **Severity:** HIGH
  - **Status:** FIXED

- **Area D: Social Connectors (YouTube Path/Session Leakage Risk)**
  - **Issue:** `app.py` performed a hacky file copy over the global `token.pickle` because it didn't pass the `user` payload into the uploader function. `subir_youtube.py` also used relative file paths which might fail outside the CWD.
  - **Files:** `app.py`, `subir_youtube.py`
  - **Functions:** `_call_youtube_uploader`, `_resolve_token_path`
  - **Root Cause:** Legacy migration debt where the app tried to shoehorn tenant tokens onto a global session file space.
  - **Severity:** MEDIUM
  - **Status:** FIXED

- **Area G: Dashboard/History (Unretrievable Content & Blind Errors)**
  - **Issue:** Generated videos were just displayed as text paths instead of actual viewable/downloadable outputs, and pipeline errors were completely hidden from the user dashboard. 
  - **Files:** `app.py`, `templates/user.html`
  - **Functions:** HTML Dashboard `.events`, `app.py` (`@app.route("/videos/<path:filename>")` created)
  - **Root Cause:** Incomplete UI wiring and missing static file serving endpoints for protected multi-tenant assets.
  - **Severity:** HIGH
  - **Status:** FIXED

- **Area I: Storage/Assets (SQLite Connection Leaks)**
  - **Issue:** SQLite connection scopes were mistakenly conflated with Python context managers (`with _conn()`), which only manage transactions. This leaves stale connection references open after every storage request, potentially locking WAL files or leaking memory.
  - **Files:** `storage.py`
  - **Functions:** `init_db`, `save_user`, `load_user`, `list_users`, `user_exists`, `delete_user`
  - **Root Cause:** Misunderstanding of `sqlite3` context managers in Python.
  - **Severity:** MEDIUM
  - **Status:** FIXED

7. NEXT-LEVEL ROADMAP
- **Reliability:** The current `sqlite` deployment relies on `threading.Lock()` coupled with `sqlite3` driver limitations. Transitioning to `SQLAlchemy` async session bounds + `aiosqlite` or PostgreSQL would eliminate blocking operations on multi-tenant writes.
- **Observability:** Move raw traceback error mapping (`ultimo_error`) into proper Sentry/Datadog instrumentation.
- **UX & Registration:** Introduce an automated Stripe/Payment gateway checkout that dynamically invokes the `/crear` logic internally, granting a zero-touch SaaS self-serve sign-up without needing manual admin provisioning.
- **Scalability:** The `scheduler_loop` resides fully within `app.py` threading. It MUST be decoupled into a proper asynchronous task worker queue (e.g., Celery or RQ backed by Redis) to prevent blocking the Gunicorn/Flask request workers during mass generation.
